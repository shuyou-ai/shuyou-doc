# 通过 ShuYou 使用 OpenCode 指南

OpenCode 是一款专为终端设计的 AI 编程智能体，通过与 ShuYou 的集成，您可以获得更多强大模型的支持，提升开发效率。

## 安装 OpenCode

OpenCode 提供多种安装方式，您可以根据自己的系统环境选择合适的方法：



```bash title="快速安装"
# 使用安装脚本（推荐）
curl -fsSL https://opencode.ai/install | bash
```

```bash title="npm/pnpm/yarn"
# 使用 npm 安装
npm i -g opencode-ai@latest

# 或使用 pnpm 安装
pnpm i -g opencode-ai@latest

# 或使用 yarn 安装
yarn global add opencode-ai@latest

# 或使用 bun 安装
bun i -g opencode-ai@latest
```

```bash title="macOS &amp; Linux"
# 使用 Homebrew 安装（macOS 和 Linux）
brew install sst/tap/opencode
```

```bash title="Windows"
# 使用 Scoop 安装
scoop bucket add extras
scoop install extras/opencode

# 或使用 Chocolatey 安装
choco install opencode
```

```bash title="Arch Linux"
# 使用 paru 安装
paru -S opencode-bin
```



<Tip title="更多安装方式">
更多详细的安装选项和说明，请参考 [OpenCode 官方文档](https://opencode.ai/docs/#install)
</Tip>

## 配置 OpenCode

### 登录并配置 ShuYou

安装完成后，您需要配置 OpenCode 以使用 ShuYou 的 API。

#### Step 1: 启动 OpenCode 并登录

首次使用时，OpenCode 会提示您进行登录配置：

```bash
opencode auth login
```

在提供商选择界面中选择 **ShuYou**：

&lt;div style="text-align: center;"&gt;
  <img src="https://cdn.marmot-cloud.com/storage/ShuYou/2025/10/30/PdPJcCv/opencode-login.jpg" alt="OpenCode 登录" />
&lt;/div&gt;

<Warning title="重要提示">
请确保选择 **ShuYou** 作为您的 API 提供商，而不是其他供应商。
</Warning>

#### Step 2: 输入 ShuYou API Key

选择 ShuYou 后，系统会提示您输入 API Key。请输入您的 ShuYou API Key（格式为 `sk-xxx`）：

&lt;div style="text-align: center;"&gt;
  <img src="https://cdn.marmot-cloud.com/storage/ShuYou/2025/10/30/nhMPtje/opencode-key.jpg" alt="输入 API Key" />
&lt;/div&gt;

<Tip title="获取 API Key">
您可以在 [ShuYou 控制台](https://shuyou.ai/settings/keys) 中获取或创建您的 API Key。
</Tip>

#### Step 3: 选择模型

配置完成后，使用 `/models` 命令查看和选择可用的模型：

```bash
# 启动 OpenCode
opencode

# 查看可用模型列表
/models
```

&lt;div style="text-align: center;"&gt;
  <img src="https://cdn.marmot-cloud.com/storage/ShuYou/2025/10/30/p7oX3KY/opencode-models.png" alt="选择模型" />
&lt;/div&gt;

## 故障排除

### 常见问题解决

#### API Key 错误
**问题**：提示 API Key 无效或未授权

**解决方案**：

- 检查您输入的 ShuYou API Key 是否正确
- 确认 API Key 是否已激活且有足够余额
- 验证 API Key 格式是否以 `sk-` 开头
- 重新运行 `opencode` 命令并重新配置
