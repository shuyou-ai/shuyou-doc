# OpenClaw 接入 ShuYou 使用指南

OpenClaw（原名 Moltbot，最初名为 Clawdbot）是一个强大的 AI 消息网关，可以将多个消息平台（WhatsApp、Telegram、Discord、Slack、Signal、iMessage 等）连接到 AI 模型。通过接入 ShuYou，您可以使用包括 GPT-5.2、Claude-4.5、Gemini-3、DeepSeek 等在内的多种模型。

<Info title="兼容性说明">
ShuYou 完全支持 OpenAI API 协议，可以通过简单配置与 OpenClaw 配合使用。

注意 OpenAI 协议的 base_url 为 `https://api.shuyou.ai/v1`。
</Info>

## 前置条件

- Node.js 22 或更高版本
- ShuYou API Key（参见下方[第 0 步](#第-0-步获取-ShuYou-api-key)）

## 接入方式

有两种方式将 ShuYou 与 OpenClaw 配合使用：

| 方式 | 适用场景 | 复杂度 |
| ---- | -------- | ------ |
| [方式一：使用 ShuYou PR](#方式一使用-ShuYou-pr) | 完整的 ShuYou 集成，自动发现模型 | 简单 |
| [方式二：手动配置](#方式二手动配置) | 稳定版本、自定义设置 | 中等 |

## 方式一：使用 ShuYou PR

使用 ShuYou 与 OpenClaw 最简单的方式是使用待合并的 [ShuYou 集成 PR #3305](https://github.com/openclaw/openclaw/pull/3305)，它提供了完整的 ShuYou 模型自动发现功能。

### 第 1 步：克隆并切换到 PR 分支

```bash
# 克隆 OpenClaw 仓库
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 切换到 PR #3305（ShuYou 集成）
git fetch origin pull/3305/head:ShuYou-integration
git checkout ShuYou-integration

# 安装依赖
pnpm install

# 构建项目
pnpm build
```

### 第 2 步：使用 ShuYou 运行引导程序

运行引导向导并选择 ShuYou 作为认证提供商：

```bash
pnpm openclaw onboard --auth-choice ShuYou-api-key
```

按照提示输入您的 ShuYou API Key。向导将自动：
- 配置 ShuYou 提供商
- 从 ShuYou API 发现可用模型
- 为您设置默认模型

### 第 3 步：验证设置

列出可用模型以验证配置：

```bash
pnpm openclaw models list
```

您应该看到带有 `ShuYou/` 前缀的 ShuYou 模型，例如：
- `ShuYou/deepseek/deepseek-chat`
- `ShuYou/openai/gpt-5.2`
- `ShuYou/google/gemini-3-pro-preview`
- `ShuYou/anthropic/claude-sonnet-4.5`

### 第 4 步：测试集成

发送测试消息以验证一切正常：

```bash
pnpm openclaw agent --local --agent main --message "你好，请只回复'嗨！'"
```

## 方式二：手动配置

如果您希望使用 OpenClaw 的稳定版本或需要更多配置控制，可以在 `openclaw.json` 配置文件中手动将 ShuYou 配置为显式提供商。

### 第 1 步：安装 OpenClaw

通过 npm 全局安装 OpenClaw：

```bash
npm install -g openclaw@latest
```

或运行引导向导设置 OpenClaw：

```bash
openclaw onboard --install-daemon
```

### 第 2 步：配置 ShuYou 提供商

将 ShuYou 提供商配置添加到您的 `~/.openclaw/openclaw.json` 文件：

```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "ShuYou": {
        "baseUrl": "https://api.shuyou.ai/v1",
        "apiKey": "sk-ss-v1-your-api-key-here",
        "api": "openai-completions",
        "models": [
          {
            "id": "deepseek/deepseek-chat",
            "name": "DeepSeek Chat",
            "reasoning": false,
            "input": ["text"],
            "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
            "contextWindow": 64000,
            "maxTokens": 8192
          },
          {
            "id": "openai/gpt-5.2",
            "name": "GPT-5.2",
            "reasoning": false,
            "input": ["text", "image"],
            "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
            "contextWindow": 200000,
            "maxTokens": 8192
          },
          {
            "id": "google/gemini-3-pro-preview",
            "name": "Gemini 3 Pro",
            "reasoning": false,
            "input": ["text", "image"],
            "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
            "contextWindow": 200000,
            "maxTokens": 8192
          },
          {
            "id": "anthropic/claude-sonnet-4.5",
            "name": "Claude Sonnet 4.5",
            "reasoning": false,
            "input": ["text", "image"],
            "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
            "contextWindow": 200000,
            "maxTokens": 8192
          }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "ShuYou/openai/gpt-5.2"
      },
      "models": {
        "ShuYou/deepseek/deepseek-chat": {},
        "ShuYou/openai/gpt-5.2": {},
        "ShuYou/google/gemini-3-pro-preview": {},
        "ShuYou/anthropic/claude-sonnet-4.5": {}
      }
    }
  }
}
```

<Warning title="重要提示">
请将 `sk-ss-v1-your-api-key-here` 替换为您的实际 ShuYou API Key。
</Warning>

### 第 3 步：添加更多模型（可选）

您可以在 `models` 数组中添加更多模型。请查看 [ShuYou 模型列表](https://shuyou.ai/models) 了解可用模型及其功能。

每个模型定义需要：

| 字段 | 说明 |
| ---- | ---- |
| `id` | ShuYou 中显示的模型 ID（例如 `openai/gpt-5.2`） |
| `name` | 模型的显示名称 |
| `reasoning` | 模型是否支持推理模式 |
| `input` | 输入类型：`["text"]` 或 `["text", "image"]` |
| `cost` | 成本配置（订阅套餐设为 0） |
| `contextWindow` | 最大上下文窗口大小（token 数） |
| `maxTokens` | 最大输出 token 数 |

### 第 4 步：验证配置

列出可用模型：

```bash
openclaw models list
```

您应该看到已配置的 ShuYou 模型：

```
Model                                      Input      Ctx      Local Auth  Tags
ShuYou/deepseek/deepseek-chat              text       63k      no    yes   configured
ShuYou/openai/gpt-5.2                      text+image 195k     no    yes   default,configured
ShuYou/google/gemini-3-pro-preview         text+image 195k     no    yes   configured
ShuYou/anthropic/claude-sonnet-4.5         text+image 195k     no    yes   configured
```

### 第 5 步：设置默认模型

设置您偏好的默认模型：

```bash
openclaw models set ShuYou/openai/gpt-5.2
```

## 使用 ShuYou 模型

配置完成后，您可以通过多种方式使用 ShuYou 模型：

### 通过 CLI Agent

```bash
# 运行快速 agent 命令
openclaw agent --local --agent main --message "用简单的话解释量子计算"
```

### 通过消息通道

配置您的消息通道（WhatsApp、Telegram、Discord 等），网关将自动使用您配置的 ShuYou 模型：

```bash
# 启动网关
openclaw gateway run

# 检查通道状态
openclaw channels status
```

### 切换模型

您可以随时切换模型：

```bash
# 设置不同的默认模型
openclaw models set ShuYou/anthropic/claude-sonnet-4.5

# 或在命令中指定模型（仅方式一）
openclaw agent --local --agent main --model ShuYou/deepseek/deepseek-chat --message "你好"
```

## 故障排除

### 常见问题

#### API Key 错误或认证失败
**问题**：出现 API Key 无效或未授权的错误

**解决方案**：

1. **检查 API Key 格式**：
   - 订阅 API Key 应以 `sk-ss-v1-` 开头
   - 按量付费 API Key 应以 `sk-` 开头
   - 确保没有多余的空格或换行符

2. **验证 API Key**：
   - 订阅：访问[订阅管理页面](https://shuyou.ai/platform/subscription)检查订阅状态和配额
   - 按量付费：访问[按量付费页面](https://shuyou.ai/platform/pay-as-you-go)确保余额充足

3. **检查配置文件语法**：
   - 确保 JSON 格式正确（无尾随逗号、正确引号）
   - 验证 API Key 正确放置在 `apiKey` 字段中


#### 模型未找到
**问题**：OpenClaw 报告找不到模型

**解决方案**：

1. **方式一（PR 分支）**：
   - 确保您在 `ShuYou-integration` 分支上（从 PR #3305 检出）
   - 拉取最新更改后运行 `pnpm build`

2. **方式二（手动配置）**：
   - 验证模型已在 `models.providers.ShuYou.models` 中定义
   - 检查模型 ID 是否完全匹配（区分大小写）
   - 同时将模型添加到 `agents.defaults.models`


#### 连接失败
**问题**：OpenClaw 无法连接到 ShuYou

**解决方案**：

- 检查网络连接是否正常
- 验证 `baseUrl` 设置为 `https://api.shuyou.ai/v1`
- 确保防火墙没有阻止出站 HTTPS 连接
- 尝试运行 `curl https://api.shuyou.ai/v1/models` 测试连通性


#### 显示缓存的旧模型
**问题**：配置更改后仍显示旧模型

**解决方案**：

1. 终止运行中的网关进程：
   ```bash
   pkill -9 -f openclaw-gateway
   ```

2. 重新构建项目（如果使用方式一）：
   ```bash
   pnpm build
   ```

3. 再次列出模型：
   ```bash
   openclaw models list
   ```
