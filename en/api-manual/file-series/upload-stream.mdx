> - OpenAI-compatible `POST /v1/files` with `multipart/form-data`
> - Upload local files (PDF, images, etc.) and receive a `file_id`
> - Reference the `file_id` in [Chat Completions](/en/api-manual/language-series/openai/create-chat-completion) or [Responses API](/en/api-manual/language-series/openai/openai-responses)
> - Alternatively pass files inline via URL or Base64 — see [Multimodal guide](/en/guide/advanced/multimodal)

## Purpose values

| `purpose` | Typical use |
| --- | --- |
| `assistants` | Files for assistant / tool workflows |
| `vision` | Images for vision models |
| `user_data` | General user-provided documents |
| `batch` | Batch API input files |
| `fine-tune` | Fine-tuning datasets |

## Example (cURL)

```bash
curl https://api.shuyou.ai/v1/files \
  -H "Authorization: Bearer $ShuYou_API_KEY" \
  -F purpose="assistants" \
  -F file="@document.pdf"
```

Use the returned `id` as `file_id` in subsequent API calls.
