# Get Subscription Detail

<Tip title="ЁЯТб Troubleshooting">
Encountering errors? See the [API Error Codes Reference](/en/guide/advanced/error-codes) for a complete list of error types and troubleshooting steps.
</Tip>

```
GET https://api.shuyou.ai/v1/management/subscription/detail
```

Returns the current account's subscription details, including plan information, Flow rates, account status, and quota usage across all time windows.

## Authentication

### Authorization Header **Required**

```http
Authorization: Bearer <ShuYou_MANAGEMENT_API_KEY>
```

- **Name**: `Authorization`
- **Format**: `Bearer &lt;API_KEY&gt;`
- **Description**: A Management API Key created in the [ShuYou Console](https://shuyou.ai/platform/management)

<Warning title="тЪая╕П Management API Key required">
This endpoint only accepts Management API Keys. Standard API Keys are not supported.
</Warning>

## Rate Limiting

Each endpoint has its own independent rate limit counter. The maximum number of requests per minute is configured at the platform level. Exceeding the limit returns a `422` error.

## Returns

### data.plan `object`

Current subscription plan details:

- `tier` `string` тАФ Plan tier: `"free"` / `"pro"` / `"max"` / `"ultra"`
- `amount_usd` `number` тАФ Monthly plan price (USD)
- `interval` `string` тАФ Billing cycle, always `"month"`
- `expires_at` `string` тАФ Current subscription period expiry time (ISO 8601)

### data.currency `string`

Currency unit. Always `"usd"`.

### data.base_usd_per_flow `number`

The platform base rate for the current plan (USD per Flow).

### data.effective_usd_per_flow `number`

The account's actual effective rate (USD per Flow). Normally equals the base rate; may be higher when usage anomalies are detected.

### data.account_status `string`

The account's current status:

| Value | Description |
|---|---|
| `healthy` | Normal |
| `monitored` | Usage anomaly detected; service remains available |
| `abusive` | Abusive usage detected; restrictions applied |
| `suspended` | Account suspended |
| `banned` | Account banned |

### data.quota_5_hour `object`

5-hour rolling window quota (high-frequency protection):

- `usage_percentage` `number` тАФ Fraction of the window consumed (0тАУ1), to 4 decimal places
- `resets_at` `string | null` тАФ Window reset time (ISO 8601); `null` if the window has not yet started
- `max_flows` `number` тАФ Maximum Flows available in this window
- `used_flows` `number` тАФ Flows consumed so far
- `remaining_flows` `number` тАФ Flows remaining
- `used_value_usd` `number` тАФ USD value of consumed Flows
- `max_value_usd` `number` тАФ USD value of the total window quota

### data.quota_7_day `object`

7-day rolling window quota. Same field structure as `quota_5_hour`.

### data.quota_monthly `object`

Monthly subscription cycle quota (upper limits only; no real-time usage data):

- `max_flows` `number` тАФ Maximum Flows available in the current billing cycle
- `max_value_usd` `number` тАФ USD value of the total monthly quota

<Info title="ЁЯТб Quota behavior">
- `quota_5_hour` and `quota_7_day` are **rolling windows** that update in real time after each request.
- `quota_monthly` is a fixed cap for the current billing cycle and does not include real-time usage.
- All three windows are enforced simultaneously тАФ hitting any one of them will rate-limit further requests.
</Info>
